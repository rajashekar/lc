## 2024-05-24 - Overly Permissive CORS Configuration
**Vulnerability:** The proxy server had a default configuration where `CorsLayer::permissive()` was unconditionally applied to the Axum router handling incoming requests. This permissive setup effectively disables Cross-Origin Resource Sharing (CORS) security mechanisms by allowing requests from any origin, which exposes the server to potential Cross-Site Request Forgery (CSRF) and unwanted access.
**Learning:** Defaulting to a permissive CORS configuration in API endpoints, specifically in proxies handling sensitive operations or authentication context, unnecessarily broadens the attack surface. It's crucial to implement the principle of least privilege in API security configurations.
**Prevention:** Make permissive CORS an opt-in feature rather than default. This ensures the proxy server remains secure for internal usage while still allowing developers to enable cross-origin access manually using a `--cors` flag when explicitly needed.
