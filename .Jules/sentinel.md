## 2024-05-24 - Unix Socket TOCTOU and Symlink vulnerabilities
**Vulnerability:** The MCP daemon UNIX socket was created directly in the config directory, suffering from Time-Of-Check to Time-Of-Use (TOCTOU) race conditions when removing the old socket or checking if the daemon is running (`if exists { remove }`). Furthermore, it was vulnerable to local symlink attacks because it didn't check if the parent directory was a symlink before binding.
**Learning:** Unix socket security requires a defense-in-depth approach. Setting permissions on the socket file itself after creation is vulnerable to TOCTOU. Instead, the socket should be placed inside a dedicated subdirectory with strict (`0o700`) permissions created *before* the socket is bound. Avoid `Path::exists()` checks before destructive operations (like `remove_file`) or state checks; instead, perform the operation and gracefully handle `NotFound` errors or rely on the primary operation (e.g., attempting a connection) to determine state.
**Prevention:** Always create Unix sockets inside a dedicated, strictly permissioned (`0o700`) subdirectory. Use `tokio::fs::symlink_metadata` to verify the directory is not a symlink. Eliminate `.exists()` checks before file removals, preferring to handle `ErrorKind::NotFound` from the removal operation itself.
