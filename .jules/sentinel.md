## 2025-05-15 - Prevent TOCTOU Symlink Vulnerability in Unix Sockets
**Vulnerability:** The MCP Daemon was binding its Unix socket directly without ensuring the parent directory had strict permissions, and it removed any existing socket without verifying if it was a symlink. This could allow a local attacker to perform a symlink attack or exploit a Time-Of-Check to Time-Of-Use (TOCTOU) race condition to gain unauthorized access to the daemon.
**Learning:** Unix sockets must be created in a secure, dedicated directory with strict permissions (`0o700`) to prevent other users from accessing or manipulating the socket file. Additionally, checking `symlink_metadata` before removing or creating files is crucial to mitigate symlink-based attacks.
**Prevention:** Always nest Unix sockets in a dedicated subdirectory created with `tokio::fs::DirBuilder` and explicit `mode(0o700)` (on Unix). Verify that the parent directory and any existing socket files are not symlinks using `tokio::fs::symlink_metadata` before proceeding with socket creation.
