## 2025-05-15 - Insecure Unix Domain Socket Permissions
**Vulnerability:** The MCP daemon's Unix domain socket (`mcp_daemon.sock`) was created with default directory permissions, which on many systems could allow unauthorized local users to connect to the daemon and execute arbitrary commands or access sensitive state. Additionally, binding the socket in a non-isolated directory without checking for symlinks exposed the service to Time-Of-Check to Time-Of-Use (TOCTOU) symlink attacks.
**Learning:** Sockets created in shared or overly permissive configuration directories (`config_dir()`) inherit or rely on the umask, which might not be sufficiently restrictive. Fixing permissions *after* socket creation introduces a race condition.
**Prevention:** To prevent TOCTOU and secure Unix sockets, always nest the socket within a dedicated parent directory explicitly created with strict `0o700` permissions *before* the socket is bound. Additionally, verify that the parent directory is not a symlink to prevent attackers from redirecting the socket.
