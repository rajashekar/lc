## 2025-04-10 - Path Traversal and TOCTOU Vulnerability in Vector Database File Operations
**Vulnerability:** The `VectorDatabase::new` and `VectorDatabase::delete_database_in_dir` functions in `src/data/vector_db.rs` accepted unvalidated `name` parameters which were used directly to construct file paths. This introduced a path traversal vulnerability. Furthermore, the `delete_database_in_dir` function was vulnerable to a Time-Of-Check to Time-Of-Use (TOCTOU) condition because it checked if a file existed using `.exists()` prior to removing it with `fs::remove_file`.
**Learning:** Checking file existence prior to an operation leaves a small window where another process can manipulate the file, circumventing the intended constraints. Also, relying implicitly on internal configuration or state to be safe against path traversal without explicit validation exposes applications to dangerous file manipulation attacks.
**Prevention:** Avoid path traversal by explicitly validating file path parts using string contains checks for path separators (`/`, `\`) and parent directory references (`..`) prior to path construction. Prevent TOCTOU issues by attempting the file system operation directly (like `fs::remove_file`) and gracefully handling the `NotFound` error condition instead of relying on a pre-check with `.exists()`.
